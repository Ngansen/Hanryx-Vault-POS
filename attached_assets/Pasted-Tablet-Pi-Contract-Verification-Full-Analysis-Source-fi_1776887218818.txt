3. Polling-loop empty → Idle Fix
✅ Correctly applied. The fix is at line 153–155:

obj.has("empty") && obj["empty"].asBoolean -> {
    if (screen !is TabletScreen.Idle) screen = TabletScreen.Idle
}
The tablet uses Gson's JsonObject (not Android's org.json.JSONObject), so there is no optBoolean — obj["empty"].asBoolean is the equivalent. The guard obj.has("empty") prevents a crash when empty is absent. Works correctly.

5. Server URL — ⚠️ HARDCODED LOCAL IP
Line 93:

var serverUrl by remember { mutableStateOf(
    prefs.getString("serverUrl", "http://192.168.86.36:8080") ?: "http://192.168.86.36:8080"
) }
The default is the local LAN IP 192.168.86.36, not the Tailscale IP 100.125.5.34. This only matters for the default — if the user has already saved the Tailscale IP in settings it's persisted in SharedPreferences under key "serverUrl" and survives restarts. But a fresh install or settings reset will default to LAN-only.

Recommended fix for the tablet agent: Change the default in two places:

// Line 93 — change both occurrences of 192.168.86.36 to 100.125.5.34
var serverUrl by remember { mutableStateOf(
    prefs.getString("serverUrl", "http://100.125.5.34:8080") ?: "http://100.125.5.34:8080"
) }
6. Additional Bugs Found
Bug A — Cloud fallback missing auth header (lines 128–132):

val cloudReq = Request.Builder()
    .url("$cloudUrl/tablet/trade/current")
    .header("Cache-Control", "no-store")
    // ❌ X-API-KEY header is NOT sent to cloud
    .get().build()
The Pi request sends X-API-KEY but the cloud fallback doesn't. If the Replit cloud server ever enforces token auth on this endpoint, the tablet will get a 401 from cloud and display authError = true (incorrectly blaming the token). Should add .header("X-API-KEY", apiToken) to the cloud request builder too.

Bug B — Server has NO auth check on /tablet/trade/current:
The server's tablet_trade_current_get() handler reads straight from the DB with no token validation — the X-API-KEY header the tablet sends is accepted but never verified. This means the endpoint is currently open to anyone who knows the URL. This is probably intentional for now (the offer data is low-sensitivity), but worth flagging.
